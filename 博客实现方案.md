# 个人博客简化方案

## 1. 项目背景与目标
- **背景**：基于现有 `Vue 3 + Vite` 前端项目扩展为个人博客站点，继续复用现有 Axios 封装及文件上传能力。
- **目标**：提供文章展示、简单评论、分组浏览以及个人后台管理的基础功能，保持简洁实用。
- **关键诉求**：
  - 文章支持 Markdown 与 PDF 两种格式。
  - 文章可配置分组与标签，访客能够快捷筛选。
  - 每篇文章跳转详情页阅读，并在底部开放简单评论（无需审核、无验证码）。
  - 个人后台可上传、编辑、发布文章以及简单维护评论。

## 2. 技术栈与依赖
- **核心框架**：Vue 3（`<script setup>`）、Vite。
- **状态管理**：Pinia。
- **路由管理**：Vue Router 4。
- **UI & 样式**：简单的 CSS 变量 + 原子类。
- **Markdown 渲染**：`markdown-it` + `highlight.js`。
- **PDF 预览**：`pdfjs-dist`（内嵌 viewer）或轻量第三方组件。
- **HTTP**：沿用 `axios` 封装（`src/api/request.js`）。
- **构建与部署**：Vite 构建、Nginx/静态托管。

## 3. 系统角色（简化）
| 角色 | 权限 | 说明 |
| --- | --- | --- |
| 游客 | 浏览文章、按分组/标签过滤、提交评论（即时展示） | 无需登录 |
| 管理员 | 全部权限：文章 CRUD、上传资源、评论管理（删除）、查看统计 | 仅站点拥有者（你自己） |

## 4. 功能模块划分
- **公共端**
  - 首页：最新文章列表、分组入口。
  - 分组/标签页：列表、筛选、分页。
  - 文章详情页：内容展示（Markdown/PDF）、评论区。
  - 关于页：个人信息、联系方式。
- **后台端**
  - 仪表盘：文章数量、阅读量统计。
  - 文章管理：列表、编辑器（Markdown/PDF 上传）、发布。
  - 分组与标签管理：简单的增删改查。
  - 评论管理：查询、删除。

## 5. 界面与导航
- **公共导航**：`首页 / 归档 / 分组 / 关于`。
- **文章详情路由**：`/posts/:slug`。
- **后台导航**：`仪表盘 / 文章 / 分组标签 / 评论`。

## 6. 关键用户流程
1. **文章浏览**：访客进入首页 → 浏览列表 → 点击文章 → 阅读 → 评论。
2. **管理员发布**：登录后台 → 新建文章 → 选择 Markdown/PDF → 上传/编辑 → 配置分组标签 → 发布。
3. **游客评论（无审核）**：访客在文章详情底部填写昵称+内容 → 直接提交 → 即时显示。
4. **管理员管理评论**：后台评论列表 → 删除不当评论。

## 7. 数据模型设计（简化）
### 7.1 表结构
- `users`: `id`, `username`, `password_hash`, `avatar_url`, `created_at`, `last_login_at`。
- `blog_articles`: `id`, `title`, `slug`, `summary`, `content_type`(`md`/`pdf`), `content_text`, `content_url`, `cover_url`, `group_id`, `tags`(JSON字符串), `status`, `publish_time`, `views`, `created_at`, `updated_at`。
- `blog_groups`: `id`, `name`, `slug`, `description`, `order_index`。
- `blog_comments`: `id`, `article_id`, `nickname`, `content`, `created_at`。

### 7.2 缓存策略（简化）
- 文章列表：Redis 缓存 10 分钟。
- 文章详情：Redis 缓存 30 分钟。

## 9. 前端实现规划
### 9.1 路由结构
```
/
  ├─ /posts
  │   └─ /posts/:slug
  ├─ /groups/:groupId
  ├─ /archive
  ├─ /about
  └─ /admin (登录守卫)
      ├─ dashboard
      ├─ articles
      │   ├─ list
      │   └─ edit/:id?
      ├─ groups
      └─ comments
```

### 9.2 Pinia Store 划分
- `useAuthStore`: Token、当前用户、登录/注销逻辑。
- `useArticleStore`: 列表缓存、详情缓存、分页状态。
- `useGroupStore`: 分组缓存。
- `useCommentStore`: 按文章维护评论列表。

### 9.3 UI 组件
- 通用组件：`NavBar`, `Footer`, `ArticleCard`, `TagList`, `Pagination`, `CommentList`, `CommentForm`, `MarkdownViewer`, `PdfViewer`。
- 后台组件：`AdminLayout`, `SideMenu`, `StatCard`, `EditForm`, `Table`。

### 9.4 文章内容渲染
- Markdown：代码高亮、图片懒加载。
- PDF：引入 `pdf.js` viewer 组件。

### 9.5 评论实现（简化）
- 游客评论提交：输入昵称（可空，默认"访客"）、内容 → API 提交 → 写入列表顶部。
- 无需验证码、无需审核、无防刷限制。

### 10.4 核心功能实现要点
- **文章管理**：
  - Markdown 文章：存储原文到 `content_text` 字段，前端渲染。
  - PDF 文章：通过 `module-file` 上传，存储 URL 到 `content_url`。
  - Slug 自动生成：使用拼音或自定义。
  
- **评论系统（简化）**：
  - 游客评论：无需登录，直接写入。
  - 无需验证码、无防刷、无敏感词过滤。
  - 单层评论（无回复层级）。

- **JWT 鉴权（简化）**：
  - 单 Token (24小时有效期)。
  - 后台接口通过 `AuthInterceptor` 验证 token。
  - 游客接口无需验证。

- **统计功能（简化）**：
  - 阅读量：文章详情接口调用时 +1。
  - 评论数：评论新增/删除时更新。

- **缓存策略（简化）**：
  - 文章列表：Redis 缓存 10 分钟。
  - 文章详情：Redis 缓存 30 分钟，更新/删除时清除。
